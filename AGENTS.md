# Tira

A password-gated app for two people to rank tiramisu places they've tried, Beli-style: three
tiers (Liked It / It Was Okay / Didn't Like It) with pairwise-comparison insertion within each
tier and a derived 0-10 score.

## Stack

- Vite + React + TypeScript, TanStack Router (file-based routes in `src/routes/`)
- Tailwind v4 + shadcn/ui (`src/components/ui/`), theme tokens in `src/index.css`
- Turso (hosted SQLite) via `@libsql/client/web`, HTTP transport - see `src/lib/db.ts`
- Leaflet + react-leaflet with OpenStreetMap tiles (no API key) for the homepage map view
- Deployed to GitHub Pages via `.github/workflows/deploy.yml`

## Key files

- `src/lib/places.ts` - all place CRUD + tier queries
- `src/lib/ranking.ts` - binary-insertion comparison state machine (new/existing/tie outcomes) +
  score bands
- `src/lib/auth.ts` - client-side password gate (SHA-256 hash comparison, not real security)
- `src/lib/geo.ts` - `coordinateFor()` resolves a place's real lat/lng when it has one, else falls
  back to a deterministic mock derived from its id, plus haversine distance (see Gotchas)
- `src/lib/place-search.ts` - Nominatim-backed place search, city geocoding, and address geocoding
  used by the add-place overlay (see Gotchas)
- `src/lib/use-geolocation.ts` - browser geolocation hook used by the list/map views and the
  add-place overlay
- `src/components/place-list-view.tsx` - homepage searchable list, sorted by score
- `src/components/place-map-view.tsx` - homepage Leaflet map with score-badge pins
- `src/components/add-place-overlay.tsx` - homepage overlay for adding a place: search or manual
  entry, tier pick, pairwise-comparison ranking, save. `/add` is now just a redirect to `/` with
  `?add=true` so old deep links/bookmarks still open it
- `src/components/ui/sheet.tsx` - radix-ui `Dialog`-based bottom sheet used by the add-place
  overlay
- `src/components/form-field.tsx` - shared form field wrapper used by the add-place overlay and
  edit flow

## Env vars (`.env.local`, gitignored)

`VITE_APP_PASSWORD_HASH`, `VITE_TURSO_DB_URL`, `VITE_TURSO_AUTH_TOKEN` - the same three are set
as GitHub Actions repo secrets for the deploy build.

## Gotchas

- `src/routeTree.gen.ts` is auto-generated by the router's Vite plugin but is committed (not
  gitignored), because `tsc` runs before `vite build` in `npm run build` and needs the file to
  already exist for a fresh CI checkout. Regenerate it locally (`npm run dev` or `npm run build`)
  before committing route changes.
- Places added via the search or manual-address flow in the add-place overlay get a real lat/lng,
  stored in `lat`/`lng` columns on the `places` table (added via `addColumnIfMissing` in
  `src/lib/db.ts`, a migration safe to re-run against an already-migrated DB). Older places, and
  manually-entered ones with no address match, have no stored coordinate, so
  `coordinateFor()` in `src/lib/geo.ts` falls back to a deterministic mock derived from the
  place's id for the map pin and "X mi" distance. This mixed-fidelity state is a known,
  intentional tradeoff, not a bug.
- `src/lib/place-search.ts` hard-restricts place search to a bounding box (`bounded=1` +
  `viewbox`) around the resolved location instead of using Nominatim's soft `bounded=0` bias -
  otherwise irrelevant same-named results from anywhere in the world can outrank real local
  matches. The overlay never falls back to an unscoped global search before a location resolves;
  it prompts the user to wait for geolocation or search a city instead.
